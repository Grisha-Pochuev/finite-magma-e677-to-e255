# Обратная цепочка известного ребра

Дата: 2026-05-19

Эта записка выделяет общий механизм, который появился при распутывании
self-типа второго уровня.

## Базовая формула

Из E677 используется форма:

```text
L_y^{-1}(x)=x*((y*x)*y)
```

Если уже известно ребро:

```text
a*z=c
```

то это означает:

```text
L_a(z)=c
```

значит:

```text
z=L_a^{-1}(c)
```

и по формуле E677:

```text
z = c*((a*c)*a)
```

То есть любое известное ребро `a*z=c` дает обратную цепочку:

```text
a*c = p
p*a = h
c*h = z
```

Коротко:

```text
a*z=c  ==>  c*((a*c)*a)=z
```

## Почему это важно

Раньше отдельные разрезы выглядели как набор удачных клеток:

```text
8*0
6*0
6*6
6*5
4*6
5*4
```

Теперь видно, что это не случайный список. Каждый известный продукт создает
следующую обратную цепочку.

## Применение к текущему self-типу case 45

Текущий self-тип:

```text
7*0=4
6*6=6
```

Здесь:

```text
b3=6
b4=5
t=4
```

Из лестницы известно:

```text
6*4=5
```

Применяем обратную цепочку к ребру `6*4=5`:

```text
6*5 = s
s*6 = u
5*u = 4
```

Это ровно маршрут:

```text
s=b3*b4
u=s*b3
b4*u=t
```

Если в ветке дополнительно:

```text
s=4
u=3
```

то получаем известное ребро:

```text
5*3=4
```

Снова применяем обратную цепочку:

```text
5*4 = v
v*5 = w
4*w = 3
```

Это объясняет, почему после маленького хвоста `u=3` следующей структурной
клеткой становится не произвольная клетка строки `5`, а именно `5*4`.

## Рабочая лемма-направление

Для дальнейшей работы полезно думать не “какая клетка самая узкая”, а:

```text
какие ребра уже известны, и какую обратную цепочку они порождают?
```

Если цепочка замыкается на запрещенный плохой параметр `r_j=b_{j+1}` или на
повтор строки, появляется противоречие или сильный разрез.

Следующая практическая цель:

```text
для текущего self-типа строить дерево не по всем клеткам, а по обратным цепочкам
из известных ребер:

6*4=5
5*3=4
...
```

Пользовательский ноутбук сейчас не нужен.

## Последний закрытый подхвост

В текущем self-типе `case 45` рассматривалась цепочка:

```text
7*0=4
6*6=6
6*5=4
4*6=3
```

Она соответствует:

```text
t=4
q=b_3*b_3=b_3
s=b_3*b_4=t
u=s*b_3=3
```

После этого известное ребро:

```text
5*3=4
```

дает следующий обратный шаг:

```text
5*4=v
v*5=w
4*w=3
```

Диагностика показала возможные значения:

```text
v=1,2,3,7,8
```

Результаты:

```text
v=2 -> противоречие
v=7 -> противоречие
v=8 -> закрыто, status none, nodes 13
v=1 -> закрыто, status none, nodes 11
v=3 -> закрыто, status none, nodes 34
```

Итого подхвост

```text
7*0=4, 6*6=6, 6*5=4, 4*6=3
```

закрыт полностью.

Остаются в ветке `6*5=4` другие значения среднего шага:

```text
4*6=1, 4*6=2, 4*6=7, 4*6=8
```

Ранее:

```text
4*6=0,4,5
```

дали противоречие.

## Релейная форма обратной цепочки

На следующем шаге полезнее использовать не только одно ребро

```text
a*z=c  ==>  z=c*((a*c)*a)
```

а его релейную форму:

```text
a*z=c и c*h=z  ==>  (a*c)*a=h
```

Объяснение: из первого ребра следует `z=c*((a*c)*a)`. Если уже известно
`c*h=z`, то из перестановочности строки `c` получаем `(a*c)*a=h`.

В текущем хвосте это дает цепочку:

```text
4*6=u
5*u=4
v=5*4
v*5=6
```

Затем еще один такой же перенос:

```text
v*5=6
6*4=5
p=v*6
p*v=4
```

Если дополнительно `v=u`, то из `p*v=4` и `4*6=v` следует еще один шаг:

```text
r=p*4
r*p=6
```

## Новые закрытые подхвосты 2026-05-20

В ветке

```text
7*0=4
6*6=6
6*5=4
```

значения `4*6=1,2,7,8` были продолжены через релейный шаг `v=5*4`.

Полностью закрыт диагональный класс:

```text
4*6=u
5*4=u
u in {1,2,7,8}
```

Проверка шла не прямым большим поиском, а по релейным следствиям
`u*5=6`, затем `p=u*6`, `p*u=4`, а в диагональном случае дальше
`r=p*4`, `r*p=6`. Все конечные подветки дали `status none`.

Также почти полностью закрыта ветка `4*6=1`:

```text
4*6=1
5*4=1,2,3,7,8 -> закрыто
```

Остался только особый хвост:

```text
4*6=1
5*4=0
```

Он особый потому, что реле возвращается в нулевую строку:

```text
5*4=0
0*5=6
0*6=7
7*0=4
```

То есть цепочка замыкается на старую 9-циклическую лестницу и не дает нового
короткого ребра автоматически. Диагностика по строке `6` оставляет 35 живых
вариантов после первого разреза, поэтому это не нужно добивать вслепую.
