# GitHub runner diagnostics — 2026-07-12

## Зачем нужен этот короткий прогон

Прогон `29168947710` показал, что конфигурация Vampire

```text
--cores 4 -m 14500
```

опасна для стандартной машины GitHub Actions с 4 виртуальными ядрами и примерно 15 ГиБ доступной физической памяти.

В текущем Vampire `-m` реализован как предел адресного пространства одного процесса. Портфельный режим создает несколько дочерних процессов. Поэтому значение `14500` нельзя считать общим бюджетом четырех работников.

## Что проверяет диагностика

Каждое из 20 коротких заданий:

1. сохраняет модель процессора, производителя, частоты, кэши, виртуализацию, память, подкачку и сведения cgroup;
2. измеряет простую пропускную способность памяти одним и четырьмя потоками;
3. запускает один короткий поиск на исходной задаче;
4. каждую секунду измеряет память всего дерева процессов;
5. сохраняет журналы и итог даже при технической ошибке.

## Сравниваемые профили

Каждый профиль повторяется четыре раза на независимо выданных машинах:

- Vampire `casc`, 4 работника, по 2500 МБ;
- Vampire `casc`, 3 работника, по 3300 МБ;
- Vampire `casc`, 2 работника, по 5000 МБ;
- Vampire `casc --intent sat`, 4 работника, по 2500 МБ;
- E prover `--auto-schedule=4`, по 2500 МБ.

Во всех профилях теоретический суммарный предел работников держится около 10 ГБ, чтобы оставить несколько гигабайт операционной системе, родительскому процессу и служебным программам.

## Ограничения интерпретации

- Короткий поиск не проверяет математическую гипотезу и не заменяет полный прогон.
- Он нужен для проверки команд, памяти, устойчивости и относительной скорости конфигураций.
- Модель процессора стандартного runner-а заранее не выбирается; прогон только измеряет, какие машины фактически выдал GitHub.

## Файлы

- `run_profile.sh` — сбор паспорта и запуск короткого профиля;
- `monitor_tree.py` — измерение памяти дерева процессов;
- `stream_bench.c` — небольшой тест пропускной способности памяти;
- `workflow-used.yml` — точная копия сценария GitHub Actions;
- `RESULTS.md` — проверенный итог после завершения.
