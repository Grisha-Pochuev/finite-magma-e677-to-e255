# Experiments

Эта папка хранит все вычислительные прогоны проекта на GitHub Actions и других внешних машинах.

## Структура

- `START_HERE_WEB.md` — постоянная память для ChatGPT в веб-чате между разными чатами.
- `YYYY-MM-DD-short-name/` — отдельная папка одного прогона.

В папке каждого прогона по возможности должны лежать:

- `README.md` — цель и устройство прогона;
- входные задачи и сценарии;
- `workflow-used.yml` — точная копия сценария GitHub Actions, который использовался;
- `RESULTS.md` — краткий проверенный итог;
- ссылки на запуск и артефакты;
- заметки о технических сбоях и о том, что нельзя считать математическим результатом.

## Правила

1. Один прогон — одна датированная папка.
2. Важный результат не должен оставаться только во временных артефактах GitHub Actions: краткий итог нужно сохранить в `RESULTS.md`.
3. Исторические сценарии сохраняются как использованные, даже если позднее в них обнаружилась ошибка.
4. Новые прогоны не должны автоматически запускаться от обычного изменения файлов. Полный запуск выполняется только после явного решения пользователя.
5. Перед большим прогоном обязателен короткий сквозной тест команд, входных файлов, журналов и загрузки артефактов.
6. Все технические выводы отделяются от математических: сбой, тайм-аут или отсутствие доказательства не равны опровержению цели.
