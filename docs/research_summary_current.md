Главный вывод:

- для оставшихся верхних веток `7*0=2,3,4,5` диагностика показывает общий

- это усилено строковой проверкой: при `7*0=2,3,4,5` соответственно

  строки `8`, начиная с `8*2`;
- это пока не полноценная лемма: после дополнительного условия `6*6=6` ветки
  `7*0=2,3,4,5` начинают расходиться.

Широкое обновление: создана записка `long_cycle_r2_classification.md`.
Главная ролевая формула для всех длинных cycles размера `9`:

`если r_2=b_2*0=t, то b_3*t=b_4`.

Значит обычные ветки должны анализироваться через строку `b_3`, прежде всего
`b_3*b_3` и `b_3*0`. Так как `t != b_3` и строка `b_3` является
перестановкой, сразу следует маленькая лемма `b_3*b_3 != b_4`. Особые
значения `r_2` образуют отдельные типы:

- `r_2=b_1`: перенос к строке `b_1`;
- `r_2=b_2`: само-тип `b_2`;

## Главный объект

Для конечной `E677`-магмы все левые умножения

`L_y(x)=y*x`

являются перестановками.

Для фиксированного `x` цель `E255` равна:

`((x*x)*x)*x = x`.

Обозначим:

`sigma(x) = (x*x)*x`.

Тогда надо доказать:

`sigma(x)*x = x`.

## Что уже нельзя использовать как путь доказательства

### Нельзя доказывать правую сократимость

Есть известная конечная `E677`-магма размера `496`, где правой сократимости нет, но `E255` выполняется.

Значит цель тоньше: столбец `x` может пропускать многие значения, но не должен пропускать именно `x`.

### Нельзя доказывать идемпотентность

Есть конечные линейные модели, например:

`x*y = 4x + y (mod 7)`,

где `E677` и `E255` выполняются, но `x*x=x` неверно для всех `x`.

## Полезные эквивалентные формы

Для фиксированного `x`, `E255` эквивалентно любому из условий:

- существует `y` такое, что `y*x=x`;
- существует `y` такое, что `(x*y)*x=y`;
- существует `y` такое, что `x*(y*x)=y`;
- существует `y` такое, что `x*(y*y)=y`.

Если решение `y*x=x` существует, оно единственно и равно:

`sigma(x)=(x*x)*x`.

Если решение `x*(y*y)=y` существует, оно также единственно и равно:

`L_x^{-4}(x)`.

## Новые рабочие ограничения

### 1. Все строки различны

Если `L_y=L_z`, то `y=z`.

Значит возможный контрпример должен иметь попарно различные строки-перестановки.

### 2. Что запрещено для плохого элемента

Пусть `a` - плохой элемент, то есть нет `y` с `y*a=a`.

Тогда для каждого `y` запрещено:

- `y*a = a`;
- `y*a = L_a^{-1}(y)`;
- `y*y = L_a^{-1}(y)`.

### 3. Цикл длины 4

Если для некоторого `x` цикл

`x -> L_x(x) -> L_x^2(x) -> L_x^3(x) -> x`

имеет длину 4, то `E255` для этого `x` не может выполняться.

Причина:

- если `E255` выполняется, то уравнение `x*(y*y)=y` имеет единственное решение;
- это решение равно `L_x^{-4}(x)`;
- при цикле длины 4 это решение должно быть `x`;
- но `x*(x*x)=L_x^2(x) != x`.

Значит правильная цель - анализировать всю карту

`F_x(y)=x*(y*y)`,

и особенно доказать невозможность собственных циклов длины 4, либо использовать такой цикл для построения контрпримера.

### 4. Коллизии в плохом столбце

Если `y*a = z*a = c`, то

`(y*c)*y = (z*c)*z`.

Если при этом `y*c=z*c`, то из левой обратимости следует `y=z`.

Значит столкновения в плохом столбце обязаны расходиться в следующем связанном столбце.

Проверка на известном примере размера 496:

- найдено `892800` пар столкновений в правых столбцах;
- во всех случаях `y*c != z*c`;
- во всех случаях `(y*c)*y = (z*c)*z`.

Значит это не случайный фильтр, а настоящий структурный мотив `E677`.

Усиление: то же верно не только для плохого столбца. Если разные строки `y` и `z` совпадают в любом столбце `x`, то при `c=y*x=z*x` обязательно

`(y*c)*y = (z*c)*z`,

и одновременно `y*c=z*c` невозможно. Это добавлено в поиск как общее правило столкновений.

Есть и обратная версия: если значение `x` стоит в строках `y` и `z` в одном и том же столбце, то

`(y*x)*y = (z*x)*z`.

Она следует из формулы `L_y^{-1}(x)=x*((y*x)*y)` и левого сокращения.

### 5. Новое правило из обратимости левого умножения

Из Lemma 13.1(i):

`L_y^{-1}(x) = x * ((y*x)*y)`.

Практическая форма:

если уже известны строки `y` и `a=y*x`, то `b=a*y` тоже известен, и тогда принудительно:

`x*b = L_y^{-1}(x)`.

Это правило добавлено в основной поиск. Оно полезно тем, что заставляет отдельные клетки строки `x` до выбора всей строки `x`.

Усиление 2026-05-13: добавлена частичная перестановочная форма этой же идеи.

Если по отдельным клеткам уже известны

- `a = y*x`;
- `c = L_y^{-1}(x)`;
- `d = L_x^{-1}(c)`;

то сразу принудительно:

`a*y=d`.
