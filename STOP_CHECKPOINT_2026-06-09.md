# Стоп-чекпойнт 2026-06-09

Работа приостановлена по просьбе пользователя. После этого чекпойнта новые
вычисления не запускались.

## Что доказано

### 1. Мостовая рекурсия обобщена на любое общее ребро

Для фиксированной цели `b` определены:

```text
A_b(q)=единственный a, для которого q*a=b;
R_b(q)=q*b.
```

Каждая строка содержит двойной интервал:

```text
A_b(q) -> b -> R_b(q).
```

Если:

```text
q_{i+1}=R_b(q_i),
a_i=A_b(q_i),
```

то следующий мост равен:

```text
a_{i+1}=(q_i*q_{i+1})*q_i
q_{i+1}*a_{i+1}=b.
```

Это доказано в:

```text
general_target_bridge_orbit_lemma.md
right_p_orbit_bridge_recursion_lemma.md
```

### 2. Одиночного свободного хвоста в конечной магме нет

Если орбита `q -> q*b` возвращается в исходное волокно источников, мостовая
цепь замыкается на исходной метке.

Если она не возвращается, то при первом повторении две различные строки
содержат одно общее ребро:

```text
b -> Q.
```

Они немедленно образуют новый двухсторонний веер: разные обратные опоры,
разные вершины впереди и один общий возвратный центр.

Это доказано в:

```text
cycle_entry_two_sided_fan_lemma.md
right_p_orbit_collision_duality_lemma.md
```

### 3. Центр нового веера не является свободным

Если повторенная вершина удовлетворяет:

```text
Q*a_Q=b,
```

то общий центр нового веера `b -> Q` равен точно:

```text
a_Q=pred_Q(b).
```

То есть при смене общего ребра старая мостовая метка становится новым общим
центром.

Доказано в:

```text
cycle_entry_hub_transport_lemma.md
```

### 4. Помеченный граф двойных интервалов

Для фиксированного `b` построен функциональный граф:

```text
q -> q*b,
```

где вершина `q` имеет метку `A_b(q)`.

Доказано:

```text
одинаковый выход графа -> разные метки;
одинаковая метка       -> разные выходы.
```

Это точная графовая форма двух видов столкновений двойных интервалов.

Файл:

```text
labeled_right_translation_graph_lemma.md
```

### 5. Good-P выровненный вход классифицирован

В good-ветви существует ровно одно ребро столбца `P`, которое обращается
строкой `P`:

```text
e*P=f
P*f=e.
```

Поэтому любой другой цикл-входной веер является действительно
невыровненным.

Доказано в:

```text
good_p_unique_reverse_edge_lemma.md
right_fixed_point_uniqueness_lemma.md
```

## Что пока не доказано

Полная No-Free-Tail лемма еще не доказана.

Точный остаток:

```text
исключить бесконечную рекурсивную смену невыровненных общих ребер

  a_0 -> a_1
  a_1 -> a_2
  a_2 -> a_3
  ...

которая каждый раз переносит центр, но никогда не возвращается к
терминальному плохому якорю r_{m-2}.
```

## Новая псевдолесная гипотеза

Для фиксированного `b` рассмотрен неориентированный граф с одним ребром на
каждую строку:

```text
A_b(q) -- R_b(q).
```

Кандидат:

```text
каждая связная компонента этого графа имеет не больше одного цикла;
эквивалентно, число ребер компоненты не превосходит число ее вершин.
```

Если это доказать, No-Free-Tail завершится сильным счетом. Для плохого `b`
обе координаты избегают `b`, поэтому `n` ребер лежат на `n-1` вершинах, что
невозможно для такого псевдолеса.

Статус:

```text
сильная гипотеза / не доказана.
```

## Свидетельства для псевдолесной гипотезы

Она проверена на:

```text
всех 496 целях известной нелинейной E677-модели размера 496;
15 линейных E677-моделях над простыми полями до 31.
```

Во всех случаях:

```text
число ребер каждой компоненты <= число вершин.
```

В модели размера 496 для каждой цели дополнительно наблюдалось:

```text
ровно одна петля;
нет противоположно направленных пар ребер.
```

Это вычислительное свидетельство, не доказательство.

Последняя точечная проверка показала:

```text
внешняя петля интервала при размере 5 невозможна
```

в полном поиске соответствующего фиксированного узла. Это также только
конечное свидетельство.

## Отвергнутый путь

Пробовался простой инъективный сертификат компоненты вида:

```text
L_b^2(q).
```

Он неверен уже в известной модели размера 496. Несколько коротких термов
работают в отдельных моделях, но общего короткого терма пока не найдено.
Не использовать их как лемму.

## Следующий правильный шаг

Не продолжать числовые проверки петель.

Нужно доказать или опровергнуть минимальную псевдолесную формулировку:

```text
две независимые циклические цепи двойных интервалов не могут лежать в одной
связной компоненте графа A_b(q)--R_b(q).
```

Предпочтительный способ:

```text
найти прямое совпадение двух полных интервалов
A_b(q) -> b -> R_b(q),
которое через two-step source reconstruction отождествит две разные строки.
```

Если этот путь не дает структурного доказательства, надо вернуться к
терминальному якорю и переносить `r_{m-2}` через один невыровненный переход,
а не запускать широкий перебор.

## Финальное уточнение перед остановкой

После основной записи проверено минимальное двухцикловое ядро графа
`A_b(q)--R_b(q)` размера 5: две петли, соединенные одним ребром. Заданные
интервалы были:

```text
3*0=2, 3*2=0
4*1=2, 4*2=1
1*0=2, 1*2=1
```

Полный точечный поиск дал `status none` почти мгновенно. Это поддерживает
псевдолесную гипотезу, но остается конечным свидетельством.

Важная коррекция: наблюдение «для каждой цели имеется ровно одна петля»
относится только к известной нелинейной модели размера 496. Оно неверно для
всех E677-магм. Например, в линейной модели над `F_5` с параметрами
`A=2, B=4` для каждой цели имеются пять петель. Они лежат в разных
одноточечных компонентах.

Следовательно, не использовать как лемму ни одно из утверждений:

```text
петля единственна;
наличие петли само по себе эквивалентно E255;
петля обязана давать противоречие.
```

Правильный остаток гипотезы глобальный:

```text
одна связная компонента не может содержать два независимых цикла.
```

Следующая содержательная цель после возобновления: разобрать минимальную
связную двухцикловую конфигурацию через полные четырехзвенные сертификаты
каждого ребра, а не продолжать считать отдельные петли.

## Поздний аудит вычислительных свидетельств

Проверки с произвольно фиксированными ролями нельзя одновременно проводить
только на канонических представителях строки `0`. Поэтому прежние точечные
закрытия фиксированных двухпетельных и других графовых ядер больше не
используются как математическое свидетельство.

Корректный режим полного перебора меток добавлен позже в
`tools/search_counterexample_strong.js` под именами `rawmodel` и
`rawcounterexample`.

Доказанные символьные леммы и проверки полных известных моделей этим аудитом
не затрагиваются.

## Дополнение при финальной паузе

После аудита найден новый, более прямой кандидат. Для направленного пути
`a->v->c` в `H_b`, заданного строками

```text
p*a=b, p*b=v,
r*v=b, r*b=c,
u=p*v,
k=u*p,
```

строго известно `v*k=b`. Во всех проверенных известных моделях элемент

```text
y=(b*c)*(u*k)
```

удовлетворяет `y*b=b` и тем самым свидетельствует E255 для `b`.

Общего доказательства из E677 нет. Этот результат записан только как
кандидат в `directed_two_edge_witness_candidate.md`.

Если кандидат удастся доказать, граф плохой цели не содержит направленных
путей длины два: каждая вершина будет чистым источником или чистым стоком.
Это сразу сожмет вынужденное двухцикловое ядро до чередующейся структуры.

Позднее доказана общая конверсия из любого `Y*b=b` в свидетель
`(b*z)*b=z`; см. `right_fixer_to_balanced_witness_lemma.md`. Поэтому
отдельный синтез `z` больше не нужен.

Прямая цель `Y=(b*c)*(u*k), Y*b=b` не закрылась двумя ограниченными слоями
ground-насыщения. Не повышать глубину механически. Основной фронт остается:
тройной веер и смешанный ядровый узел `2+1` с полными сертификатами.
