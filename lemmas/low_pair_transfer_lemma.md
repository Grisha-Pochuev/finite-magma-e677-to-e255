```text
low_to_low_role_lemma.md
```

Этот файл остается подробным журналом открытия low-pair/low-to-low формы.
Для текущего продолжения читать сначала `low_to_low_role_lemma.md`.

## Зачем создан этот файл

После закрытия cross-pair

```text
{6*6,6*5}={7,8}
```

следующий остаток в ветке `7*0=4` состоит из низких значений `6*5`.

Текущая карта:

```text
6*6=8: closed 6*5 in {1,3,7}; open 6*5 in {2,4}
6*6=7: closed 6*5=8; open 6*5 in {1,2,3,4}
```

Новый вопрос: являются ли значения `1,2,3,4` одним типом, или они делятся на
пары `1/3` и `2/4`.

## Диагностика пары `1/3` при `6*6=7`

Для

```text
7*0=4
6*6=7
6*5=1
```

получено:

```text
row 6 domain: 190
6*0 in {2,3,4,6,8}
```

Для

```text
7*0=4
6*6=7
6*5=3
```

получено:

```text
row 6 domain: 151
6*0 in {1,2,4,6,8}
```

В обоих случаях активной остается строка `6`, а главный маркер тот же:

```text
u=(6*5)*6=6*0
q=u*6
6*(0*q)=0
```

## Первый контроль

Проверен специальный узел `u=6`. Тогда `q=6*6=7`, поэтому релейный прогноз
требует:

```text
6*8=0
```

Результаты:

```text
6*6=7, 6*5=3, u=6 -> status none, about 1s
6*6=7, 6*5=1, u=6 -> status none, about 1s
```

Это говорит, что значения `1` и `3` действительно ведут себя как один
низкий тип, по крайней мере на первом контрольном узле.

## Диагностика пары `2/4`

Для

```text
7*0=4
6*6=7
6*5=2
```

получено:

```text
row 6 domain: 155
6*0 in {1,3,4,6,8}
```

Для

```text
7*0=4
6*6=7
6*5=4
```

получено:

```text
row 6 domain: 164
6*0 in {1,2,3,6,8}
```

Контрольный узел `u=6` снова закрылся сразу:

```text
6*6=7, 6*5=2, u=6 -> status none, about 1s
6*6=7, 6*5=4, u=6 -> status none, about 1s
```

## Общий диагностический шаблон

Для всех низких значений

```text
s=6*5 in {1,2,3,4}
```

в слое `6*6=7` наблюдается одна форма:

```text
6*0 in {1,2,3,4,6,8} \ {s}
```

и специальное значение

```text
u=6
```

закрывается мгновенно через:

```text
q=6*6=7
6*8=0
```

## Текущий статус

Это еще не доказанная лемма. Это новый рабочий кандидат:

```text
low-pair transfer:
низкие значения 6*5 снова передают давление в строку 6 через
u=(6*5)*6=6*0 и q=u*6.
```

## Следующий шаг

Следующий осмысленный шаг - не закрывать все значения механически, а проверить
следующий общий маркер:

```text
u=8
```

Он присутствует во всех четырех низких ветках и является естественным
партнером уже закрытого cross-pair значения `8`.

Минимальная проверка:

```text
6*6=7, 6*5=s, 6*0=8, s*6=8
q=8*6
6*(0*q)=0
```

Сначала проверить один представитель, например `s=2` или `s=3`.

## Проверка общего маркера `u=8`

Проверен представитель:

```text
7*0=4
6*6=7
6*5=2
u=2*6=6*0=8
```

Диагностика строки `8`:

```text
q=8*6 in {0,1,2,6}
```

Полный `q`-слой закрылся:

```text
q=0 -> require 6*1=0 -> status none
q=1 -> require 6*2=0 -> status none
q=2 -> require 6*3=0 -> status none
q=6 -> require 6*7=0 -> status none
```

Итог:

```text
6*6=7
6*5=2
u=8
status: closed
```

Структурный смысл: общий маркер `u=8` продолжает тот же механизм

```text
q=u*6
6*(0*q)=0
```

и не рождает нового глубокого остатка.

## Точка остановки

Работа приостановлена после закрытия первого представителя `u=8`.

Следующий шаг при продолжении:

```text
проверить u=8 для s=1,3,4
```

Если они повторят результат `s=2`, можно считать `u=8` вторым устойчивым
слоем low-pair transfer после мгновенно закрытого `u=6`.

## Обновление 2026-05-24: `u=8` повторился на паре `1/3`

Проверка продолжена не как механическое добивание всех низких значений, а как
тест переносимости общего маркера `u=8`.

Для:

```text
7*0=4
6*6=7
6*5=1
u=1*6=6*0=8
```

диагностика строки `8` снова дала:

```text
q=8*6 in {0,1,2,6}
```

Весь `q`-слой закрылся:

```text
q=0 -> require 6*1=0 -> status none, time 13.88s, nodes 11
q=1 -> require 6*2=0 -> status none, time 16.56s, nodes 51
q=2 -> require 6*3=0 -> status none, time 24.26s, nodes 410
q=6 -> require 6*7=0 -> status none, time 11.61s, nodes 1
```

Для:

```text
7*0=4
6*6=7
6*5=3
u=3*6=6*0=8
```

диагностика также дала:

```text
q=8*6 in {0,1,2,6}
```

и весь `q`-слой закрылся:

```text
q=0 -> require 6*1=0 -> status none, time 13.02s, nodes 8
q=1 -> require 6*2=0 -> status none, time 18.88s, nodes 267
q=2 -> require 6*3=0 -> status none, time 19.87s, nodes 681
q=6 -> require 6*7=0 -> status none, time 10.25s, nodes 1
```

Итого на этом этапе маркер `u=8` был подтвержден на трех низких представителях:

```text
s=1 -> closed by q-layer
s=2 -> closed by q-layer
s=3 -> closed by q-layer
```

Значение `s=4` специально не добивалось на этом шаге, чтобы не превращать
проверку в механическое закрытие всех комнат. Текущий вывод: `u=8` ведет себя
как устойчивый второй слой low-pair transfer после мгновенно закрытого `u=6`.

Следующий правильный шаг - сформулировать ролевую low-pair подлемму:

```text
при 6*6=7 и низком s=6*5 значение u=8
не создает нового глубокого остатка;
оно всегда сводится к одному q-слою
q=8*6, 6*(0*q)=0.
```

После такой формулировки можно либо проверить `s=4` как финальный sanity check,
либо подняться уровнем выше и обновить карту оставшихся типов в `7*0=4`.

## Обновление 2026-05-24: `u=8` закрыт для всех низких `s`

После формулировки повторяющейся формы `s=4` был использован только как
финальный sanity check, а не как начало нового механического добивания.

Для:

```text
7*0=4
6*6=7
6*5=4
u=4*6=6*0=8
```

диагностика строки `8` снова дала тот же слой:

```text

```

Весь `q`-слой закрылся:


```text

s=3 -> closed by q-layer
s=4 -> closed by q-layer
```

Ролевая формулировка локального слоя:

```text
case45, b1=8, b2=7, b3=6, b4=5

b2*0=b5
b3*b3=b2
b3*b4=s, где s in {b5,b6,b7,b8}
b3*0=b1

Тогда слой q=b1*b3 сводится к q in {0,b8,b7,b3}
и каждый вариант закрывается условием b3*(0*q)=0.
```

В числах это ровно:

```text
q=8*6 in {0,1,2,6}
6*(0*q)=0
```

Статус этого слоя:

```text
low-pair transfer / u=b1 layer -> local closed
```

Следующий открытый вопрос уже не `u=8`, а оставшиеся low-to-low переносы:

```text
6*6=7
6*5=s in {1,2,3,4}
u=s*6=6*0 in {1,2,3,4} \ {s}
```

Их нельзя добивать как список комнат. Следующий шаг должен искать новый
ролевой маркер для low-to-low transfer, вероятно сравнивая пары `1/3` и `2/4`.

## Обновление 2026-05-24: low-to-low слой получил двухслойную форму

После предыдущей диагностики проверялись не все хвосты подряд, а только
представители двух ролей:

```text
compact targets: u=1=b8 and u=4=b5
wide targets:   u=2=b7 and u=3=b6
```

Во всех проверках использовалась одна и та же релейная цепочка:

```text
s=6*5
u=s*6=6*0
q=u*6
6*(0*q)=0
```

### Компактные целевые строки

Для компактных целей слой `q` закрывается прямо:

```text
s=2,u=1:
  q=0 -> closed
  q=1 -> closed
  q=2 -> closed
  q=6 -> closed

s=2,u=4:
  q=0 -> closed
  q=1 -> closed
  q=2 -> closed
  q=7 -> closed
```

### Широкие целевые строки

Для широких целей почти весь `q`-слой тоже закрывается прямо:

```text
s=1,u=2:
  q=0 -> closed
  q=1 -> closed
  q=2 -> closed
  q=6 -> closed

s=1,u=3:
  q=0 -> closed
  q=1 -> closed
  q=2 -> closed
  q=6 -> closed
```

Единственное отличие широкого слоя - дополнительное значение:

```text
q=7=b2
```

В этом случае появляется следующий маркер:

```text
r=6*7 = b3*b2
```

и диагностика дает точную форму:

```text
r in {1,2,3,4} \ {s,u}
```

Проверенные закрытия:

```text
s=1,u=2,q=7,r=3 -> closed
s=1,u=2,q=7,r=4 -> closed

s=4,u=2,q=7,r=1 -> closed
s=4,u=2,q=7,r=3 -> closed

s=1,u=3,q=7,r=2 -> closed
s=1,u=3,q=7,r=4 -> closed

s=4,u=3,q=7,r=1 -> closed
s=4,u=3,q=7,r=2 -> closed
```

## Текущая кандидат-лемма

```text
Low-to-low transfer closes by a two-layer relay.

Layer 1:
  u=s*6=6*0
  q=u*6
  6*(0*q)=0

Layer 2, only for wide targets u=2,3 and q=7:
  r=6*7
  r in {1,2,3,4} \ {s,u}
  each r branch closes.
```

Ролевой смысл:

```text
compact u=1,4:
  row u has no persistent extra q-tail.

wide u=2,3:
  the only persistent extra q-tail is q=b2;
  it is controlled by the next backward marker b3*b2.
```

Следующий шаг уже не должен быть механическим закрытием всех низких пар.
Нужно переписать эту форму в обозначениях `b_j`, а затем использовать
оставшиеся low-to-low представители как sanity check.

## Обновление 2026-05-24: перенос на соседний слой `6*6=8`

После формулировки двухслойной формы был проверен соседний слой:

Компактный представитель снова закрывается на `q`-слое:

```text
  q=7,r=1 -> closed
```

Расширенная кандидат-лемма:

```text
The low-to-low relay does not depend on whether 6*6 is 7 or 8.
It depends on the target role of u=6*0=s*6:

endpoint target -> closes at q-layer;
interior target -> only q=7 remains, then closes at r=6*7.
```

## Ролевая запись

В чистом 9-цикле case45:

```text
b1=8
b2=7
b3=6
b4=5
b5=4
b6=3
b7=2
b8=1
```

Текущий низкий блок:

```text
Low = {b5,b6,b7,b8} = {4,3,2,1}
endpoints = {b5,b8} = {4,1}
interior = {b6,b7} = {3,2}
```

Рабочая форма:

```text
b2*0=b5
b3*b4=s, with s in Low
b3*0=s*b3=u, with u in Low \ {s}
q=u*b3
b3*(0*q)=0
```

Если `u` является endpoint в `Low`, проверенные представители закрываются на
слое `q`. Если `u` является interior в `Low`, единственный устойчивый хвост:

```text
q=b2
r=b3*b2
r in Low \ {s,u}
```

и проверенные представители закрываются на слое `r`.
