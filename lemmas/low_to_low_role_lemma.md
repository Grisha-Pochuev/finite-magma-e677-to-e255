# Low-to-Low Role Relay

Дата: 2026-05-24.

## Статус

Статус этой записи:

```text
candidate / local finite certificate
```

Это не общая доказанная теорема E677. Это ролевой кандидат, который объясняет
проверенные низкие остатки в ветке `case45`, `7*0=4`.

## Ролевая нотация

В полном 9-цикле:

```text
0 -> 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 8 -> 0
```

используем:

```text
b1=8
b2=7
b3=6
b4=5
b5=4
b6=3
b7=2
b8=1
```

Текущий низкий блок:

```text
Low = {b5,b6,b7,b8} = {4,3,2,1}
endpoints = {b5,b8} = {4,1}
interior = {b6,b7} = {3,2}
```

## Проверяемая зона

Рабочая зона:

```text
case45
7*0=4
b2*0=b5
b3=6
b4=5
```

Низкий перенос:

```text
s=b3*b4=6*5, где s in Low
u=s*b3=s*6
b3*0=6*0=u
```

После этого первый релейный маркер:

```text
q=u*b3=u*6
b3*(0*q)=0
```

В числах:

```text
q=u*6
6*(0*q)=0
```

## Доказанная цепочка до первого маркера

Эта часть является не эвристикой, а прямым следствием E677.

В ветке `7*0=4` лестница дает:

```text
b3*b5=b4
```

В числах:

```text
6*4=5
```

Берем низкое значение:

```text
s=b3*b4
u=s*b3
```

По обратной цепочке E677 из ребра `b3*b5=b4` следует:

```text
b4*u=b5
```

В числах:

```text
5*u=4
```

Так как строка `0` содержит полный цикл:

```text
0*b5=b4
```

и уже есть:

```text
b4*u=b5
```

релейная форма дает:

```text
b3*0=u
```

В числах:

```text
6*0=u
```

Теперь применяем E677 при `x=0, y=b3`:

```text
0 = b3*(0*((b3*0)*b3))
```

Поскольку:

```text
b3*0=u
q=u*b3
```

получаем:

```text
b3*(0*q)=0
```

В числах:

```text
6*(0*q)=0
```

Именно это является доказанным ядром low-to-low relay.

## Ручное сжатие возможных `q`

Из условия:

```text
b3*(0*q)=0
```

следует, что колонка `0*q` в строке `b3` должна быть свободной для значения
`0`.

Но в рассматриваемом узле строка `b3` уже содержит:

```text
b3*b5=b4
b3*b4=s
b3*0=u
b3*b3=d, где d=b2 или d=b1
```

Поэтому колонка `0*q` не может быть одной из:

```text
b5, b4, 0, b3
```

Так как строка `0` является полным циклом, это запрещает:

```text
q in {b6,b5,b1,b4}
```

В числах:

```text
q notin {3,4,8,5}
```

Остается общий ручной верхний список:

```text
q in {0,b8,b7,b3,b2}
```

В числах:

```text
q in {0,1,2,6,7}
```

Это объясняет, почему все диагностические `q`-слои были маленькими. Вычисления
используются уже не для поиска широкого домена `q`, а для различения:

```text
endpoint target u
interior target u
```

## Найденная роль целевого `u`

Проверки показали, что ключевая разница не в исходном `s`, а в роли целевого
значения:

```text
u endpoint in Low -> q-layer closes directly
u interior in Low -> only extra q=b2 remains
```

То есть:

```text
u in {b5,b8} = {4,1}
  closes at q-layer

u in {b6,b7} = {3,2}
  q-layer closes all values except q=b2=7
```

## Второй слой для wide/interior target

Если `u` является interior и появляется хвост:

```text
q=b2=7
```

то включается второй маркер:

```text
r=b3*b2=6*7
```

Диагностика и проверки дают:

```text
r in Low \ {s,u}
```

и все проверенные ветки `r` закрываются.

## Проверенные представители в слое `6*6=7`

Компактные цели:

```text
s=2,u=1 -> q in {0,1,2,6} -> closed
s=2,u=4 -> q in {0,1,2,7} -> closed
```

Широкие цели:

```text
s=1,u=2:
  q=0,1,2,6 -> closed
  q=7 -> r=3,4 -> closed

s=4,u=2:
  q=0,1,2,6 -> closed
  q=7 -> r=1,3 -> closed

s=1,u=3:
  q=0,1,2,6 -> closed
  q=7 -> r=2,4 -> closed

s=4,u=3:
  q=0,1,2,6 -> closed
  q=7 -> r=1,2 -> closed
```

Субагентская сверка 2026-05-24 подтвердила: для wide-целей `u=2,3` в слое
`6*6=7` действительно использовался `r`-слой при `q=7`.

## Проверка переносимости в слой `6*6=8`

Тот же рисунок повторился в соседнем слое:

```text
6*6=8
```

Особое значение:

```text
u=7
```

закрывается прямо:

```text
6*6=8,s=2,u=7 -> closed
6*6=8,s=4,u=7 -> closed
```

Компактные цели:

```text
6*6=8,s=2,u=1 -> q-layer closed
6*6=8,s=2,u=4 -> q-layer closed
```

Широкие цели:

```text
6*6=8,s=2,u=3:
  q=0,1,2,6 -> closed
  q=7 -> r=1,4 -> closed

6*6=8,s=4,u=2:
  q=0,1,2,6 -> closed
  q=7 -> r=1,3 -> closed
```

## Кандидат-лемма

Рабочая формулировка:

```text
Low-to-low relay:

Если в ветке case45, 7*0=4,
s=b3*b4 лежит в Low,
и u=s*b3=b3*0 тоже лежит в Low \ {s},
то закрывающий разрез определяется ролью u:

endpoint u:
  q=u*b3 and b3*(0*q)=0 closes directly;

interior u:
  q=u*b3 and b3*(0*q)=0 closes except q=b2;
  the remaining q=b2 closes by r=b3*b2 in Low \ {s,u}.
```

## Что пока является сертификатом, а не ручным доказательством

Доказанная вручную часть:

```text
s=b3*b4
u=s*b3
=> b3*0=u
q=u*b3
=> b3*(0*q)=0
```

Пока проверено конечными сертификатами:

```text
endpoint u -> q-layer closes;
interior u -> only q=b2 remains;
q=b2 -> r=b3*b2 closes.
```

Следующий теоретический шаг, если продолжать именно лемму:

```text
объяснить вручную, почему для interior u единственный устойчивый хвост именно
q=b2, и почему тогда r=b3*b2 попадает в Low \ {s,u}.
```

До этого момента статус остается:

```text
candidate / local finite certificate
```

## Почему это важно

Раньше низкие значения выглядели как список отдельных комнат:

```text
6*5=1,2,3,4
```

Теперь они имеют ролевую структуру:

```text
source s in Low
target u=s*b3=b3*0
endpoint / interior role of u
q-layer
optional r-layer
```

Это снижает риск закапывания: следующий шаг должен быть не новым перебором, а
обновлением карты всей ветки `7*0=4` с учетом этой ролевой подлеммы.
