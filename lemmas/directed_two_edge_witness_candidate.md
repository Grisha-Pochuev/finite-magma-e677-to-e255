# Кандидат: направленный путь из двух ребер дает свидетель E255

Статус:

```text
сильный кандидат / не доказан
```

## Конфигурация

Для фиксированной цели `b` пусть в графе `H_b` есть направленный путь
из двух соседних ребер:

```text
a -> v -> c.
```

Пусть первое ребро задается строкой `p`, а второе строкой `r`:

```text
p*a=b,  p*b=v,
r*v=b,  r*b=c.
```

Из полного сертификата первого ребра введем:

```text
u=p*v,
k=u*p=pred_v(b).
```

Тогда строго следует:

```text
v*k=b.
```

## Наблюдаемый свидетель

В известных моделях выражение

```text
y=(b*c)*(u*k)
```

удовлетворяет:

```text
y*b=b.
```

По эквивалентности Lemma 13.2(vi) это немедленно давало бы E255 для `b`.
Поэтому желаемая общая лемма имеет вид:

```text
если в H_b есть направленный путь a -> v -> c,
то b удовлетворяет E255.
```

Эквивалентная форма E255 использует элемент `z`, для которого

```text
(b*z)*b=z.
```

Однако отдельный синтез `z` больше не нужен. Общая доказанная лемма

```text
right_fixer_to_balanced_witness_lemma.md
```

строит такой `z` из любого доказанного равенства `Y*b=b`.

## Что проверено

Формула `y=(b*c)*(u*k)` проверена:

```text
- на всех направленных соседних парах ребер для цели 0
  в известной нелинейной E677-модели размера 496;
- на выборках для целей 0,1,17,31,255,495 той же модели;
- на линейных E677-моделях, использованных в текущем аудите.
```

В тех же выборках работали также:

```text
(b*r)*(b*v)
(b*u)*(c*k).
```

На объединенной выборке из направленных путей модели размера 496 было
проверено `1440` экземпляров смешанной конфигурации. Во всех них три
выражения

```text
(b*r)*(b*v)
(b*c)*(u*k)
(b*u)*(c*k)
```

фиксировали `b` справа. В проверенных моделях значение
`(b*c)*(u*k)` также совпадало с каноническим фиксатором
`(b*b)*b`. Это наблюдение, а не выведенное тождество.

Это только модельное свидетельство. Все проверенные модели уже удовлетворяют
E255, поэтому совпадение может использовать скрытое следствие E255 и не обязано
выводиться из одного E677.

## Последний линейный расчет перед обрывом

Для совместного синтеза была выбрана линейная E677-модель над `F_7`:

```text
y*x = 4y+x  (mod 7).
```

Для направленного пути условия дают:

```text
p*b=v,
r*v=b,
r=2(b-v)  (mod 7).
```

Уравнение желаемого свидетеля

```text
(b*z)*b=z
```

в этой модели имеет решение:

```text
z=-b  (mod 7).
```

Это задает обязательную линейную координату для синтеза терма на прямом
произведении с моделью размера 496.

Простой кандидат

```text
z=(v*b)*r
```

не прошел общую проверку на направленных путях модели размера 496 и должен
быть отброшен.

Последний запуск синтеза оборвался только из-за отсутствующего временного
имени `op7` в интерактивной сессии. Это не математический результат. При
возобновлении нужно заново определить:

```text
mod7(x)=x mod 7,
op7(y,x)=4y+x mod 7.
```

Нелинейная модель размера 496 полностью воспроизводится из:

```text
tools/check_known_non_right_cancellative_example.js
```

Поэтому потерянное временное состояние интерактивной сессии не содержит
незаменимых данных.

После восстановления был проведен полный семантический синтез всех различных
термов глубины не более двух над переменными

```text
b,v,p,r,c,u,k
```

на `144` совместных примерах `F_7 x M_496`. Ни один терм, использующий данные
пути, не удовлетворял `(b*z)*b=z` на всей выборке. Это отрицательное
ограничение только для данной глубины, не математическая невозможность.

Затем найдено и доказано общее преобразование. Если `Y*b=b`, положить:

```text
t=Y*((b*Y)*b),
z=t*b.
```

Тогда:

```text
(b*z)*b=z.
```

Следовательно, дальнейший синтез `z` не нужен.

## Аудит короткого символьного вывода

Проведено ограниченное насыщение равенств, использующее только:

```text
- шесть заданных клеток направленного пути;
- исходное E677;
- общую Lemma 13.1(iii);
- конгруэнтность операции;
- левое сокращение.
```

Первый слой использовал все константы и их произведения глубины один:

```text
56 подстановочных термов;
21763 ground-терма;
6278 исходных равенств.
```

После насыщения цель `Y*b=b` не была получена.

Во втором слое были явно добавлены `Y`, `Y*b` и ближайшие термы полных
сертификатов:

```text
62 подстановочных терма;
26693 ground-терма;
7694 исходных равенства.
```

После конгруэнтного замыкания и левого сокращения цель снова не была
получена.

Это не опровержение кандидата. Точный вывод:

```text
если доказательство существует в этой форме, ему нужен более глубокий
промежуточный терм или существенно глобальный конечный аргумент.
```

Поскольку все модельные примеры уже удовлетворяют E255, дальнейшее
увеличение глубины без новой структурной идеи будет модельной подгонкой.

## Структурное значение

Если кандидат будет доказан, то для плохого `b` в `H_b` не может быть вершины
с одновременно входящим и исходящим ребром. Тогда каждая вершина является
либо чистым источником, либо чистым стоком, а вынужденное двухцикловое ядро
из `bad_target_core_fan_lemma.md` превращается в чередующуюся сеть
источников и стоков.

Это значительно сильнее прежнего запрета crossed double fan и напрямую
сжимает оба оставшихся типа ядрового узла: тройной веер и смешанный `2+1`.

## Точная точка продолжения

1. Не запускать новый широкий поиск.
2. Не продолжать синтез отдельного `z`: эта задача решена общей леммой
   `right_fixer_to_balanced_witness_lemma.md`.
3. Единственная активная цель:

   ```text
   Y=(b*c)*(u*k)
   Y*b=b.
   ```

4. Искать символьный вывод этого равенства из E677 и двух полных
   сертификатов соседних ребер.
5. До такого вывода не объявлять Directed Two-Edge Witness леммой.
6. Не повышать глубину ground-насыщения механически. Вернуться к кандидату
   только при появлении конкретного промежуточного тождества.
