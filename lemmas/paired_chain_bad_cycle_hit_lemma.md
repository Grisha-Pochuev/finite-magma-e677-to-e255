# Paired Chain Bad-Cycle Hit Lemma

Date: 2026-06-05.

Status:

```text
proved pressure consequence / paired recursion role
```

Purpose:

```text
Classify what happens when either paired predecessor chain first hits a
bad-cycle element b_j.
```

This is the paired-recursion version of:

```text
first_return_row_pressure_lemma.md
```

## Setup

Use the paired predecessor chains:

```text
b_2*A_i=A_{i+1}
b_3*B_i=B_{i+1}.
```

They are generated by:

```text
A_{i-1}=A_i*(A_{i+1}*b_2)
B_{i-1}=B_i*(B_{i+1}*b_3).
```

The bad-cycle ladder gives, for every bad-cycle element `b_j`:

```text
b_j*r_{j-1}=b_{j+1}.
```

## Hit In The A Chain

Suppose:

```text
A_i=b_j.
```

Since:

```text
b_2*A_i=A_{i+1}
```

the edge-triangle / zipper identity gives:

```text
A_i*(A_{i+1}*b_2)=A_{i-1}.
```

Substituting `A_i=b_j`:

```text
b_j*(A_{i+1}*b_2)=A_{i-1}.
```

Together with the bad-cycle ladder:

```text
b_j*r_{j-1}=b_{j+1},
```

row `b_j` now has two forced edges:

```text
b_j*r_{j-1}=b_{j+1}
b_j*(A_{i+1}*b_2)=A_{i-1}.
```

Therefore:

```text
A_{i-1}=b_{j+1}
  => A_{i+1}*b_2=r_{j-1}
```

by injectivity of row `b_j`.

This is descent to an older bad-cycle tail.  Otherwise, row `b_j` gains an
extra occupied output.

## Hit In The B Chain

Suppose:

```text
B_i=b_j.
```

Since:

```text
b_3*B_i=B_{i+1},
```

the zipper identity gives:

```text
B_i*(B_{i+1}*b_3)=B_{i-1}.
```

Substituting `B_i=b_j`:

```text
b_j*(B_{i+1}*b_3)=B_{i-1}.
```

Together with:

```text
b_j*r_{j-1}=b_{j+1},
```

row `b_j` has:

```text
b_j*r_{j-1}=b_{j+1}
b_j*(B_{i+1}*b_3)=B_{i-1}.
```

Therefore:

```text
B_{i-1}=b_{j+1}
  => B_{i+1}*b_3=r_{j-1}.
```

This is again descent.  Otherwise, row `b_j` gains extra occupied pressure.

## Consequence

A first hit of either paired predecessor chain into the bad-cycle block is
never neutral:

```text
hit b_j
=> bad-cycle descent
   OR extra occupied pressure in row b_j.
```

Shared-edge strengthening:

```text
bad_cycle_shared_edge_descent_lemma.md
```

If the new zipper edge in row `b_j` matches any row-`0` cycle edge:

```text
b_{k+1} -> b_k,
```

then the following output must differ from `b_{k-1}` and its return column
lands exactly at:

```text
r_{k-1}=b_{k-1}*0.
```

Therefore every exact row-`0` edge match is already an explicit tail descent.
The genuinely residual occupied-row branch must avoid all row-`0` directed
edges, not merely avoid output collisions.

Thus the hard branch can exclude all bad-cycle hits and focus only on chains
whose new points are fresh relative to:

```text
the bad-cycle block,
the other chain,
row-t pressure,
row-b_4 pressure,
self-loop/self-swap roles.
```
