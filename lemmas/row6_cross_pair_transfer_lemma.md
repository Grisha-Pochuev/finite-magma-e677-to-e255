# Row 6 Cross-Pair Transfer Lemma

Дата: 2026-05-24.

## Зачем создан этот файл

После закрытия нескольких слоев внутри

```text
7*0=4
6*6=8
```

стало ясно, что продолжать просто добивать `6*5=2,4` слишком локально.
Нужен более широкий объект, который объясняет парные слои:

```text
6*6=8, 6*5=7
6*6=7, 6*5=8
```

Это не случайные значения: `7` и `8` образуют соседнюю пару вокруг строки `6`
в полном 9-цикле.

## Проверенная сторона A

Узел

```text
7*0=4
6*6=8
6*5=7
```

закрыт полностью.

Разбор:

```text
u=7*6=6*0
u in {1,2,3}
```

Статус:

```text
u=1 -> closed
u=2 -> closed, including secondary self-type q=2
u=3 -> closed by q-layer
```

Особый трудный хвост:

```text
u=2
q=2
2*6=2
```

закрылся леммой `secondary self-type relay lemma`:

```text
s=6*2
h=s*6
2*h=0
r=2*0
r*2=pred0(h)
```

## Проверенная сторона B

Проверен зеркальный слой:

```text
7*0=4
6*6=7
6*5=8
```

Диагностика:

```text
row 6 domain: 218
6*0 in {1,2,3,4,6}
```

Строка `8` остается широкой:

```text
row 8 domain: 16955
```

Главный вывод: первая активная строка снова `6`, а не строка партнера `8`.
Это повторяет сторону A, где при `6*6=8, 6*5=7` строка `6` сжалась до `224`,
а маркером стал `6*0`.

После этого сторона B закрыта первым `u,q`-слоем:

```text
u=8*6=6*0
q=u*6
6*(0*q)=0
```

Проверки:

```text
u=6 -> q=7, require 6*8=0 -> status none

u=1 -> q in {0,1,2,6} -> all status none
u=2 -> q in {0,1,2,6,7} -> all status none
u=3 -> q in {0,1,2,6,7} -> all status none
u=4 -> q in {0,1,2,7} -> all status none
```

Итог:

```text
7*0=4
6*6=7
6*5=8
status: closed
```

## Кандидат-лемма

В ветке

```text
7*0=4
6*4=5
```

если строка `6` получает cross-pair:

```text
6*6=a
6*5=b
```

где `{a,b}={7,8}`, то правильный разрез:

```text
u=b*6=6*0
q=u*6
6*(0*q)=0
```

а не прямое дробление по строке `a` или `b`.

## Текущий статус

```text
6*6=8, 6*5=7 -> closed
6*6=7, 6*5=8 -> closed
```

## Статус кандидат-леммы

Для пары

```text
{6*6, 6*5}={7,8}
```

в ветке `7*0=4` обе ориентации закрываются через один и тот же разрез:

```text
u=(6*5)*6=6*0
q=u*6
6*(0*q)=0
```

Это уже не эвристика для одной стороны, а локальная cross-pair лемма для
значений `7` и `8` внутри текущей ветки `7*0=4`.

## Следующий широкий шаг

Не возвращаться к старым хвостам. Следующий уровень - восстановить карту
оставшихся значений `6*5` при `6*6=7` и `6*6=8` и искать общий тип для
остатков:

```text
6*6=8: closed 6*5 in {1,3,7}; open 6*5 in {2,4}
6*6=7: closed 6*5=8; open 6*5 in {1,2,3,4}
```

Новый вопрос: являются ли значения `6*5 in {1,2,3,4}` одним и тем же
`low-pair transfer` типом, или `2/4` образуют отдельную симметричную пару.
