# Relay Graph Lemma - Case 45 Exploration

Дата: 2026-05-21.

## Основное релейное правило

Из E677 уже используется обратная цепочка:

```text
a*z=c  ==>  z=c*((a*c)*a)
```

Если дополнительно известно:

```text
c*h=z
```

то из перестановочности строки `c` получаем релейную форму:

```text
a*z=c и c*h=z  ==>  (a*c)*a=h
```

В текущем хвосте `case 45` это работает как граф переходов между известными
ребрами, а не как независимый перебор клеток.

## Рабочая зона

Основной self-хвост:

```text
7*0=4
6*6=6
6*5=4
```

Здесь известное ребро `6*4=5` дает:

```text
s=6*5
u=s*6
5*u=4
```

В подветке `s=4`:

```text
4*6=u
5*u=4
v=5*4
v*5=6
p=v*6
p*v=4
```

Если `v=u`, появляется еще один диагональный слой:

```text
r=p*4
r*p=6
```

## Типы переходов

### 1. Diagonal

Форма:

```text
4*6=u
5*4=u
```

Тогда:

```text
u*5=6
p=u*6
p*u=4
r=p*4
r*p=6
```

Статус:

```text
u=1,2,7,8 -> закрыто
```

Вывод: diagonal-тип является устойчивым закрывающим типом внутри текущего
self-хвоста.

### 2. Zero-return

Форма:

```text
5*4=0
```

Сначала кажется, что реле вернулось в нулевую строку:

```text
0*5=6
0*6=7
7*0=4
```

Но следующий полезный шаг берется из ребра `5*4=0` и уже известного `0*3=4`:

```text
h=5*0
h*5=3
```

Статус:

```text
4*6=1, 5*4=0 -> закрыто
4*6=2, 5*4=0 -> закрыто
4*6=7, 5*4=0 -> закрыто
4*6=8, 5*4=0 -> закрыто
```

Вывод: zero-return не является тупиком; он переводит работу к клетке `5*0`.

### 3. Senior-row Transfer

Форма:

```text
v=1,2,3,7,8
v*5=6
p=v*6
p*v=4
```

После этого диагностика почти всегда возвращает малый список по строке `6`,
а иногда коротко добивается строками `1`, `7` или `8`.

Вывод: это не самостоятельная лемма, но устойчивый вычислительно-структурный
маршрут внутри self-хвоста.

### 4. Active-row Collapse

Форма:

```text
после релейных следствий первая активная строка = 6
```

Типичный результат:

```text
first row 6
большая часть вариантов сразу мертва
остаток закрывается короткими проверками
```

Этот тип закрыл последние хвосты:

```text
4*6=7, 5*4=8, p=3/4
4*6=8, 5*4=1,2,3
```

### 5. Self-type

Исходный трудный режим:

```text
q=6*6=6
```

Он труден потому, что контрольное соотношение

```text
q*((6*q)*6)=6
```

при `q=6` вырождается. Поэтому выход из self-типа идет не через `6*q`, а через
следующее ребро:

```text
6*4=5
6*5=s
s*6=u
5*u=4
```

## Что закрыто этой схемой

В ветке:

```text
7*0=4
6*6=6
6*5=4
```

закрыты все значения среднего шага:

```text
4*6=0 -> contradiction
4*6=1 -> closed
4*6=2 -> closed
4*6=3 -> closed
4*6=4 -> contradiction
4*6=5 -> contradiction
4*6=7 -> closed
4*6=8 -> closed
```

Итого:

```text
6*5=4 -> закрыто полностью
```

## Проверка переносимости

Проверены соседние верхние ветки:

```text
7*0=2
7*0=3
7*0=5
```

Для `7*0=2` и `7*0=3` первая часть общей структуры сохраняется:

```text
6*2=5  или  6*3=5
6*6=6
```

Но подтип `s=6*5=t` не дает такого же сжатия, как в `t=4`. Строки `2` и `3`
остаются очень широкими:

```text
t=2, s=t: row 2 domain около 74322
t=3, s=t: row 3 domain около 66478
```

Для `7*0=5` ветка особая:

```text
6*5=5
```

и строка `5` включается рано, но это также не дает мгновенной общей леммы.

## Итог разведки

Результат типа B:

```text
релейная схема полезна и сильна внутри текущего self-хвоста case 45,
но пока не является общей леммой для всех соседних верхних веток.
```

Следующий общий разрез должен объяснить, почему в `t=2,3` строки `2` и `3`
остаются широкими после `s=t`, тогда как в `t=4` реле быстро сжимается через
строки `4,5,6,8`.

## Дополнение 2026-05-23: перенос в соседние `6*6=7/8`

После закрытия всего self-узла

```text
7*0=4
6*6=6
```

проверена переносимость релейного слоя на соседние значения `6*6`.

Главное уточнение: цепочка

```text
6*4=5
6*5=s
s*6=u
5*u=4
6*0=u
q=u*6
6*(0*q)=0
```

не использует само условие `6*6=6`. Она использует только ребро `6*4=5` и
полный цикл строки `0`. Поэтому это уже более общий релейный слой для ветки
`7*0=4`.

Диагностика подтвердила перенос:

```text
6*6=7, 6*5=8 -> row 6 domain 218 после фиксации s
6*6=8, 6*5=7 -> row 6 domain 224 после фиксации s
```

Но следующий слой не закрывает все сразу. В узле

```text
6*6=8
6*5=7
7*6=1
6*0=1
```

получился новый компактный остаток:

```text
q=1 -> row 6 domain 13
q=2 -> row 6 domain 16
```

Значит статус релейной схемы стал точнее:

```text
u-transfer является общей локальной леммой для ветки 7*0=4,
но для соседних 6*6=7/8 нужен следующий инвариант компактного остатка.
```
