# U-Transfer / No-Tail Trap Lemma

Дата: 2026-05-23.

Этот файл фиксирует локальную лемму, которая закрыла узел
`7*0=4, 6*6=6` внутри `case 45`.

## Закрытый фронт

Рабочая ветка:

```text
case 45
row 0 = 1 2 3 4 5 6 7 8 0
7*0=4
6*4=5
6*6=6
```

В этой ветке строка `6` имеет домен `2376`, но клетка `6*5` уже сжата:

```text
6*5 in {1,2,3,4,7,8}
```

Все возможные значения клетки `6*5` в этом узле:

```text
s=6*5 in {1,2,3,4,7,8}
```

закрыты. Значение `6*5=4` было закрыто отдельной self-swap веткой, остальные
значения закрыты через `u-transfer / no-tail trap`.

## Ролевое обозначение

```text
b3=6
b4=5
b5=4

b3*b5=b4      то есть 6*4=5
b3*b3=b3      то есть 6*6=6
b3*b4=s       то есть 6*5=s
```

Следующий неслучайный разрез:

```text
u=s*b3
```

В числах:

```text
u=s*6
```

## Главная цепочка

Если

```text
6*4=5
6*5=s
s*6=u
```

то обратная форма E677 дает:

```text
5*u=4
```

Поскольку в полном 9-цикле:

```text
0*4=5
0*5=6
```

из релейного правила следует:

```text
6*0=u
```

То есть `u=s*6` одновременно является значением клетки `6*0`. Это делает `u`
структурным маркером, а не произвольным вычислительным разрезом.

Далее вводим:

```text
q=u*6
```

Тогда E677 при `x=0,y=6` дает:

```text
6*(0*q)=0
```

В case 45 строка `0` является полным 9-циклом, поэтому `0*q` всегда остается
внутри того же цикла. Это и есть no-tail trap: ноль обязан появиться в
конкретной колонке строки `6`; он не может уйти во внешний хвост, потому что
внешнего хвоста нет.

## Кандидат-лемма

В полном 9-цикле case 45 конфигурация

```text
6*4=5
6*6=6
6*5=s
s*6=u
```

запирает ветку в строке `6` через два маркера:

```text
6*0=u
q=u*6
6*(0*q)=0
```

Локальная лемма:

```text
для s in {1,2,3,7,8} эта пара маркеров либо сразу дает столкновение
в строке 6, либо сводит ветку к короткому q-слою; все такие q-слои закрыты.
```



Для `s=1` проверены первые `u`-узлы:

```text

u=2 -> ok, затем все q-узлы закрыты
u=3 -> ok, затем все q-узлы закрыты
u=4 -> ok, затем все q-узлы закрыты
```

Для `s=1,u=3`:


Таким образом уже закрыто:

```text

6*5=1, 1*6=8
```

## Обновление 2026-05-22: ветка `s=1` закрыта полностью

Продолжение по протоколу `u-transfer / no-tail trap` закрыло оставшиеся
значения `u` для `s=1`.

Для `s=1,u=4`:

```text
q=0,3,4,5,6,8 -> contradiction
q=1 -> status none
q=2 -> status none
q=7 -> status none
```

Для `s=1,u=8`:

```text
q=3,4,5,6,7,8 -> contradiction
q=0 -> status none
q=1 -> status none
q=2 -> status none
```

Итог:

```text
6*5=1 -> закрыто полностью
```

Структурный смысл: для `s=1` маркер `u=s*6=1*6` сначала принуждает
`6*0=u`, затем `q=u*6` принуждает `6*(0*q)=0`. Этот `q`-слой оставляет не
больше трех живых значений `q`, и все они закрываются коротко. Это
подтверждает, что текущая лемма работает как настоящий разрез, а не как
случайная удачная клетка.

диагностика тоже указывала на активную строку `6`.

## Дополнение 2026-05-23: перенос есть, но появляется новый остаток

Проверка соседних узлов показала: первый слой `u-transfer` действительно не
зависит от self-условия `6*6=6`. Он следует из ребра `6*4=5`, поэтому
переносится на `6*6=7` и `6*6=8`.
